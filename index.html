HTML

  • HTML elements work together to add structure to a webpage.
  • html, head, meta, title, body are examples of elements used in HTML
  • These elements have distinct roles in the framework of a webpage!
  • !DOCTYPE is a declaration, not case sensitive.
  • Elements visible on the page are located in the 'body' section of the code.
    • Elements inside of other elements are called 'child elements'.
    In such case, 'body' is the 'parent element'.
    This labeling is used by developers to explain the relationship among elements in HTML
  • An 'element' refers to the entire section, from opening to closing tag.
    A 'tag' is what appears inside the angle brackets.

CSS

  • CSS (Cascading Style Sheets) are used to add style to webpages.
  • Defines colors, visual layout, organization, and general aesthetics of a webpage.
    • Used to define 'style rules' for specific elements, using a wide range of different options.
  • CSS can be used inline, but most commonly, external CSS style sheets are used to style webpages.
    • using a 'link' element, rel="stylesheet" href="directory here"
  • Separation of concerns:
    Separating code into different files based on use case. This is a key design principle!
  • CSS RULE
    div (selector) {property: value; property: value;}(declations)
  • There are lots of properties which can be changed! VSCode has plenty of shortcuts to help!

JavaScript

  • A high level dynamic programming language.
    adds additional interactivity to websites
    featues such as effects, data communication, and more!
  • "Every time you do something with a webpage, such as click a button, retrieve data, or fill out a form, JavaScript is at work!"
  • link to HTML by using 'script src="directory"'
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • 'const' = variables which cannot be reassigned
  • Conditional statements and loops are used to control the order in which code is executed.
    'if' statements - basic decision making for code, can interrupt basic control flow (top to bottom)
    code will only execute 'if' conditions are met (and 'else if' until meeting no conditions)
    '===' is a strict equality operator
  • use /* and */ to comment out code
